PUSH VÀ CLONE CODE REPOSITORY GITHUB STEP BY STEP

TẠO DỰ ÁN MỚI & PUSH CODE

  1. Tạo dự án repository mới trên Github (đặt tên và create)
  2. Tạo dự án trên máy tính dùng để code và lưu trữ cục bộ (New folder)
  3. Khởi tạo git: để khởi tạo một kho lưu trữ git cục bộ - Local repository
    • Mở terminal trên VS Code
    • Config mặc định chọn Git bash
    • git status : để kiểm tra trạng thái git
    • git init : khởi tạo một kho lưu trữ Local
  4. add thư mục dự án vào staging area chờ commit
    • git add . : add tất cả thư mục
    • git add tên_thư_mục : add lẻ thư mục, tệp
  5. Commit và đẩy code lên GitHub
    • git commit -m "điền nội dung commit vào đây" : sau khi commit code đẩy vào local repository chờ push lên GitHub
    • git branch -M main : do trên GitHub là main mà trong máy tính cục bộ là master nên cần cập nhật lại
    • git remote add origin Đường_dẫn_đến_dự_án (Copy link repository trên Github) : Remote đến máy chủ git
    • git push -u origin main : push code lên máy chủ Git

PUSH CODE LÊN DỰ ÁN CÓ SẴN

  1. git remote add origin Đường_dẫn_đến_dự_án
  2. git branch -M main
  3. git push -u origin main

CLONE CODE TỪ REPOSITORY KHÁC VỀ MÁY

  • git clone Đường_dẫn_đến_dự_án

Git Branches Explained

Tạo, chuyển đổi và quản lí các nhánh dự án

  1. git branch : kiểm tra cục bộ nhánh git trên máy tính
  2. git branch -a : kiểm tra nhánh trên github và máy tính dấu * ở phía nhánh nào tức là đang đứng ở nhánh đó
  3. git branch -c tên_nhánh_mới : tạo ra 1 nhánh mới nhưng vẫn ở nhánh hiện tại, chỉ tạo ở máy cục bộ
  4. git checkout tên_nhánh : qua nhánh tên_nhánh
  5. git checkout -b tên_nhánh_mới : tạo ra nhánh mới và qua nhánh đó luôn, chỉ tạo ở máy cục bộ
  6. git push -u origin tên_nhánh : push tên_nhánh lên github
  7. git branch -d tên_nhánh : không đứng ở tên_nhánh, xóa tên_nhánh khỏi máy cục bộ
  8. Nếu tên_nhánh đang thực hiện commit nhưng vẫn muốn xóa tên_nhánh ta thực hiện: git branch -D tên_nhánh
  9. git push -d origin tên_nhánh : xóa tên_nhánh khỏi github
